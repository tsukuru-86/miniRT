## 🧠 **miniRT — My first RayTracer with MiniLibX**

### **概要**

Ray Tracing の世界への導入として、C言語でシンプルなレイトレーサーを作成します。
完成後は、数式を恐れずに扱えるようになり、美しい画像を生成できるようになります。

---

## 📘 Chapter I — Introduction

3D画像生成には大きく分けて2つの方法があります：

* **Rasterization（ラスタライズ）**：高速で多くのグラフィックエンジンが採用
* **Ray Tracing（レイトレーシング）**：計算コストが高いが高品質なリアリズムを実現

miniRTでは、C言語で基本的なレイトレーサーを自作します。
目標は、数学的・物理的な式を自分で実装できるようになることです。

## ⚙️ Chapter II — Common Instructions

* **言語**：C
* **Norm準拠必須**
* **異常終了禁止**（segfault, double free など）
* **メモリリーク禁止**
* **Makefile 要件**：

  * ルール：`all`, `clean`, `fclean`, `re`, `bonus`
  * フラグ：`-Wall -Wextra -Werror`
  * 無駄な再リンク禁止
* **libft使用可**
* **テストコードの作成推奨**
* **提出物**：Gitリポジトリ内のもののみ評価対象

## 🧩 Chapter III — Mandatory part

### **Program name**

`miniRT`

### **Turn-in files**

全てのソースファイル

### **Arguments**

シーンファイル（拡張子：`.rt`）

### **Allowed functions**

* `open, close, read, write, printf, malloc, free, perror, strerror, exit`
* `math.h`内の全関数（`-lm`を付けてリンク）
* `MiniLibX`の全関数

### **Description**

レイトレーシングで画像を生成し、ウィンドウに描画する。
シーンはカメラ位置・オブジェクト・照明などで構成される。

#### **要件：**

* `MiniLibX`を使用（OS同梱 or ソース版）
* ウィンドウ操作はスムーズであること
* **最低限実装すべき形状：**

  * Plane（平面）
  * Sphere（球体）
  * Cylinder（円柱）
* 内部・交差計算を正確に扱うこと
* オブジェクトの変換をサポート：

  * 移動（translation）
  * 回転（rotation）
* **光源処理**：

  * アンビエントライト（ambient）
  * 拡散光（diffuse）
  * 影（hard shadows）
  * 完全な暗闇は禁止

#### **操作**

* `ESC`キー：終了
* ウィンドウの×ボタン：終了
* `MiniLibX`の`image`使用推奨

---

### **Scene (.rt) file format**

| 要素                    | フォーマット                                             | 説明                       |
| --------------------- | -------------------------------------------------- | ------------------------ |
| **A (Ambient light)** | `A 0.2 255,255,255`                                | 照度比(0.0–1.0), RGB(0–255) |
| **C (Camera)**        | `C -50.0,0,20 0,0,1 70`                            | 座標, 正規化ベクトル, FOV(0–180°) |
| **L (Light)**         | `L -40.0,50.0,0.0 0.6 10,0,255`                    | 座標, 輝度比(0.0–1.0), RGB    |
| **sp (Sphere)**       | `sp 0.0,0.0,20.6 12.6 10,0,255`                    | 中心座標, 直径, RGB            |
| **pl (Plane)**        | `pl 0.0,0.0,-10.0 0.0,1.0,0.0 0,0,225`             | 点座標, 法線ベクトル, RGB         |
| **cy (Cylinder)**     | `cy 50.0,0.0,20.6 0.0,0.0,1.0 14.2 21.42 10,0,255` | 中心, 軸ベクトル, 直径, 高さ, RGB   |

#### **ルール**

* `.rt`形式必須
* 各要素は改行・空白で区切り可
* 順序は任意
* 大文字識別子（A, C, L）は1回のみ
* ファイルエラー時は `"Error\n"` とメッセージを出力して終了

---

### **Example scene**

```
A 0.2 255,255,255
C -50,0,20 0,0,1 70
L -40,0,30 0.7 255,255,255

pl 0,0,0 0,1.0,0 255,0,225
sp 0,0,20 20 255,0,0
cy 50.0,0.0,20.6 0,0,1.0 14.2 21.42 10,0,255
```

## 🌈 Chapter IV — Bonus part

ボーナスは **必須部分が完璧な場合のみ** 採点対象。

### **実装例：**

* **鏡面反射**（specular reflection）
* **チェッカーパターン**（color disruption）
* **複数ライト・多色ライト**
* **二次曲面**（cone, hyperboloid, paraboloid）
* **バンプマッピング**（bump mapping）

### **ボーナスの自由度**

* `.rt`フォーマットを拡張してOK
* 他関数の使用もOK（説明できること）
